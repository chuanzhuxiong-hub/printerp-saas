## 1. 项目背景

PrintERP 是一个面向 3D 打印电商商家的 SaaS 经营管理系统。

核心场景包括：

- 多店铺电商经营管理
- 拼多多订单、账单、售后、推广、快递数据导入
- 3D 打印产品、SKU、模型、BOM、成本管理
- 耗材采购、批次成本、库存流水
- 生产任务、打印失败、成品入库
- 发货、售后、补发、退款
- 每日、每周、每月利润统计
- AI 标题优化、主图和详情页制作工作流
- 竞品监控和自动选品
- 拓竹打印机 / 打印农场集成
- 多租户 SaaS 商业化

长期产品文档：

- `PrintERP_Commercial_PRD_Optimized_v2.md`
- `PrintERP_Commercial_PRD_Optimized_AI_Ecommerce.md`
- `PrintERP_Product_Framework.md`
- `UI_STYLE_GUIDE.md`，如存在
- `README.md`

在执行任何较大任务前，请优先阅读本文件和相关产品文档。

## 2. 总体开发原则

1. 不要一次性大范围重构。
2. 每次只处理一个明确模块。
3. 修改前先阅读相关文件和产品文档。
4. 涉及多文件、多模块、重构、测试、上线前检查时，必须使用 Superpowers 工作流。
5. 小文案、小样式、小字段修改可以直接处理，不需要完整 Superpowers 流程。
6. 复杂任务先输出实现计划，不要直接改代码。
7. 等用户确认计划后，再开始修改。
8. 每次修改后必须说明：修改文件、修改原因、测试方式、业务影响、回滚风险。

## 7. 多租户数据隔离规则

PrintERP 是 SaaS 系统，必须保证不同商家的数据完全隔离。

规则：

1. 所有业务表必须包含 `tenantId`。
2. 所有业务查询必须带 `tenantId` 条件。
3. 不能通过前端传入的 `tenantId` 直接信任用户。
4. `tenantId` 应从当前登录用户的会话、token 或 tenant context 中获取。
5. 商家员工只能访问所属商家的数据。
6. 平台管理员和商家后台必须区分权限。
7. 不允许跨租户读取、修改、删除数据。
8. 所有批量导入、批量更新、批量删除必须限制在当前 `tenantId` 内。
9. 测试中必须覆盖跨租户隔离场景。

## 8. 财务和利润计算规则

所有金额相关逻辑必须严谨。

规则：

1. 所有金额字段必须使用 Decimal / numeric，不允许使用 float。
2. 前端展示金额可以格式化，但后端计算必须使用 Decimal。
3. 订单利润必须可追溯到订单实收金额、商品成本、耗材成本、包装成本、快递成本、平台佣金、支付手续费、广告成本、售后成本、补发成本和退款金额。
4. 订单利润发生相关成本变化时必须重新计算。
5. 售后、补发、退款必须影响订单最终利润。
6. 不允许用临时字段覆盖真实财务数据。
7. 报表应区分按下单时间统计和按结算时间统计。
8. 拼多多订单表不能直接等于最终利润，必须结合账单、售后、推广、快递等数据。

核心公式：

```text
订单毛利 = 实收金额 - 产品生产成本 - 快递成本 - 包装成本 - 平台佣金 - 支付手续费

订单净利 = 订单毛利 - 售后成本 - 广告成本

经营净利润 = 所有订单净利 - 固定成本 - 其他费用
```

## 9. 库存规则

库存是系统核心，禁止随意修改。

规则：

1. 所有库存变化必须生成 `inventory_transactions`。
2. 不允许直接修改库存数量而不记录流水。
3. 库存变化必须使用数据库事务。
4. 库存流水必须记录 tenantId、库存对象类型、SKU / 耗材 / 包装材料、变化前数量、变化数量、变化后数量、业务来源、来源单号、操作人和时间。
5. 采购入库、生产消耗、生产入库、销售出库、补发出库、退货入库、报废、盘点调整都必须记录流水。
6. 库存预警必须基于真实库存和警戒线。
7. 成品库存、耗材库存、包装库存应统一在库存中心管理。

## 10. 产品中心设计规则

产品中心是 PrintERP 的核心页面。

页面原则：

1. 产品是 SPU。
2. SKU 是真实销售、库存、生产、成本和利润核算对象。
3. SKU 不单独做一级菜单。
4. SKU 必须集成在产品中心，通过产品行展开 / 收起管理。
5. BOM / 打印配方不单独做一级菜单，应集成在 SKU 详情或产品详情中。
6. 模型文件、AI 标题、AI 主图、AI 详情页、竞品链接，也应集成在产品中心。
7. 产品详情优先使用 Drawer、Tabs、展开行，减少页面跳转。

产品中心应集成：

- 产品基础信息
- SKU 管理
- 模型文件
- BOM / 打印配方
- 成本设置
- 库存状态
- 利润表现
- AI 标题优化
- AI 主图制作
- AI 详情页制作
- 竞品监控
- 自动选品转产品草稿

## 11. 订单中心设计规则

订单中心应更接近电商后台，而不是普通数据库表。

规则：

1. 订单详情应集成订单商品、成本明细、生产记录、发货记录、售后记录和利润明细。
2. 发货不应作为独立一级菜单。
3. 售后不应作为独立一级菜单。
4. 发货、售后、生产应在订单详情中可查看和处理。
5. 订单列表应支持状态 Tabs：全部、待生产、生产中、待发货、已发货、售后中、已完成、已取消。
6. 订单状态必须使用统一 Badge。
7. 订单金额、成本、利润字段必须右对齐。

库存中心应统一管理，不要拆成多个一级页面。

库存中心 Tabs：

- 耗材库存
- 成品库存

- 库存流水

库存中心应支持搜索、筛选、分页、低库存高亮、批次成本展示、库存流水查看、库存预警处理和导出。

生产中心应支持 3D 打印业务特点。

应包含：

- 打印任务
- 打印机状态
- 打印失败
- 质检
- 成品入库

生产状态：

- 待生产
- 打印中
- 待质检
- 已完成

打印失败必须记录原因、耗材损耗和责任归属。

## 14. 拼多多数据导入规则

拼多多利润统计不能只导订单表。

如果统计某个月利润，建议导入：

1. 订单明细表
2. 对账中心 / 货款明细表
3. 售后退款表
4. 推广费用表
5. 快递费用表
6. 商品 / SKU 表

导入规则：

1. 订单号是第一匹配字段。
2. 运单号是物流成本匹配字段。
3. SKU ID / 商品 ID 是商品匹配字段。
4. 重复导入必须可识别。
5. 导入前必须预览。
6. 导入后必须显示成功数量、失败数量、未匹配数量。
7. 不允许重复计算同一笔费用。
8. 导入动作必须记录日志。

## 15. AI 电商增长功能规则

AI 功能应集成在产品中心，不要拆成很多一级菜单。

功能包括：

- AI 标题优化
- AI 主图制作工作流
- AI 详情页制作工作流
- 竞品链接管理
- 竞品价格动态
- 竞品销量展示值 / 估算值
- 自动选品
- 选品机会评分
- 一键转产品草稿

竞品数据规则：

1. 每个产品最多绑定 5 个竞品链接。
2. 竞品销量必须区分 `salesDisplayValue`、`salesEstimate`、`salesActual`。
3. 不能把估算销量当成真实销量。
4. 数据来源必须记录。
5. 抓取或采集必须遵守平台规则和法律合规要求。
6. 第一阶段优先支持手动录入、表格导入和低频半自动分析。

## 16. 打印农场集成规则

打印农场集成应采用安全、合规、稳定的架构。

推荐架构：

```text
PrintERP SaaS
   ↓
本地 Agent / 局域网服务
   ↓
Bambu Connect / LAN Mode / 官方或合规接口
   ↓
拓竹打印机 / 打印农场
```

规则：

1. 不要依赖非公开云接口作为核心商业功能。
2. 不要绕过拓竹认证。
3. 不要伪装官方客户端。
4. 不要把破解协议作为主要方案。
5. 优先使用局域网、Developer Mode、Bambu Connect 或官方支持路径。
6. 多地农场应通过 VPN、路由器组网或本地 Agent 同步状态。
7. SaaS 中应显示打印机状态、任务状态、失败原因和生产记录。
8. 打印任务状态变化应能同步到生产任务和库存成本。

## 17. UI / UX 规则

技术栈：

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui

不要更换 UI 技术栈。

整体风格：干净、专业、现代、适合长期办公、适合 3D 打印电商商家。

UI 规则：

1. 左侧固定侧边栏。
2. 顶部 Header 显示当前页面、面包屑、搜索、快捷操作和用户信息。
3. 数据卡片使用统一 MetricCard。
4. 状态使用统一 StatusBadge。
5. 表格使用统一 DataTable。
6. 复杂编辑优先使用 Drawer。

9. 加载状态必须使用 Skeleton。
10. 删除操作必须二次确认。
11. 成功或失败必须有 Toast。
12. 金额、库存、利润字段右对齐。
13. 低库存、低利润、未配置 BOM 必须明显提示。

一级菜单建议：

- 首页
- 产品中心
- 订单中心

- 库存中心
- 采购中心

## 24. 面向用户的产品原则

PrintERP 的目标用户是 3D 打印电商商家，不是专业 ERP 用户。

因此：

1. 页面要简单直接。
2. 术语要尽量通俗。
3. 操作路径要短。
4. 新手要有引导。
5. 出错要能看懂。
6. 报表要让老板一眼看懂赚不赚钱。
7. 成本和利润必须能追溯。
8. 用户不应该被迫理解复杂财务系统才能使用。

## 25. 最终目标

PrintERP 的最终目标是：

```text
帮助 3D 打印商家看清楚每个订单、每个 SKU、每台打印机、每个店铺到底赚不赚钱。
```

任何开发、优化、重构和 UI 改造，都应该服务于这个目标。
