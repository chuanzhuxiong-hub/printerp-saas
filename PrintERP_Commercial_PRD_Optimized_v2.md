# PrintERP 商业化优化版产品开发文档（AI 电商增长增强版）

> 本文档是在原始 `PrintERP_Product_Framework.md` 基础上优化而来，用于放入 Codex / 项目根目录，作为产品需求、信息架构、页面设计、数据库设计、开发顺序和商业化落地的长期参考文档。  
> 优化原则：**更符合 3D 打印电商商家的真实使用习惯；能集成到一起的功能不单独拆页面；先自用跑通，再 SaaS 商业化；产品中心同时承载“产品管理 + 电商内容 + 竞品监控 + 自动选品”。**

---

## 0. 本版核心优化结论

原始文档的功能覆盖比较完整，但页面和模块偏“传统 ERP”，容易造成菜单太多、操作路径太长。商业落地版本应采用：

```text
少菜单 + 强详情页 + 展开行 + 标签页 + 抽屉弹窗 + 批量导入
```

新的产品设计重点：

1. **产品管理合并 SKU / BOM / 模型 / 成本 / 成品库存**
2. **订单管理合并发货 / 售后 / 利润 / 生产任务**
3. **库存管理合并耗材 / 包装 / 成品 / 流水 / 预警**
4. **采购管理合并供应商 / 采购单 / 批次入库 / 付款记录**
5. **生产管理合并打印任务 / 打印机 / 失败记录 / 成品入库**
6. **数据导入中心统一处理拼多多订单、账单、售后、推广、快递表**
7. **首页看板变成老板每天打开就能看懂的经营驾驶舱**
8. **SaaS 后台独立给平台管理员，普通商家不看到复杂后台**
9. **第一阶段不追求全自动控制打印机，先把利润和库存算准**
10. **第二阶段加入 Bambu / 本地 Agent / 打印机状态同步**
11. **产品中心新增标题优化、主图/详情页 AI 制作工作流**
12. **每个产品最多绑定 5 个竞品链接，跟踪竞品价格、销量展示值、活动和内容变化**
13. **产品中心新增自动选品池：按关键词搜索机会品，结合价格、销量、竞争、毛利和 3D 打印可生产性给出建议**

一句话：

> PrintERP 不应做成传统“功能菜单堆叠型 ERP”，而应做成“3D 打印电商老板每天用来算利润、管生产、看库存、处理售后的经营系统”。

## 1. 产品定位升级

### 1.1 产品名称

```text
PrintERP
```

### 1.2 产品定位

PrintERP 是面向 3D 打印电商商家、小型打印农场、定制工作室的云端 SaaS 经营管理系统。

它不是普通进销存，也不是单纯打印机控制软件，而是：

```text
订单利润管理 + 3D 打印生产管理 + 耗材批次成本 + 库存预警 + 售后损失分析
```

### 1.3 核心价值

帮商家回答这 8 个问题：

1. 今天到底赚了多少钱？
2. 哪个订单赚钱，哪个订单亏钱？
3. 哪个 SKU 利润高，哪个 SKU 应该涨价或下架？
4. 哪卷耗材、哪个批次、哪个颜色成本更高？
5. 哪台打印机效率高、失败率低？
6. 售后、补发、发错货、打印失败亏了多少钱？
7. 成品、耗材、包装库存够不够？
8. 多个拼多多店铺、线下订单、私域订单能不能统一算账？
9. 产品标题、主图、详情页是否足够适合电商转化？
10. 竞品有没有降价、上活动、销量变化？
11. 哪些关键词值得开发新产品？哪些产品适合 3D 打印低成本切入？

### 1.4 商业宣传语

```text
让每一个 3D 打印订单赚不赚钱，一眼看清。
```

备选：

```text
专为 3D 打印商家打造的利润、生产、库存一体化管理系统。
```


### 2.2 用户类型

| 用户类型 | 典型痛点 | 重点功能 |
|---|---|---|
| 拼多多 3D 打印卖家 | 低价竞争、利润薄、售后多 | 订单利润、拼多多导入、SKU 成本 |
| 多店铺商家 | 多个平台账算不清 | 多店铺利润、渠道报表 |

| 线下 + 线上商家 | 手工订单和平台订单分散 | 统一订单、统一库存 |

## 3. 产品设计原则

### 3.1 能合并的，不拆菜单

错误做法：

```text
产品管理
SKU 管理
BOM 管理
模型管理
成品库存
SKU 利润
```

正确做法：

```text
产品管理
  └ 展开 SKU
      ├ 基础信息
      ├ 模型
      ├ BOM / 打印配方
      ├ 成本
      ├ 库存
      └ 利润
```

### 3.2 老板优先

老板每天最关心：

```text
销售额
利润
亏损订单
售后损失
库存预警
待发货
待生产
```

所以首页必须先显示经营结果，而不是复杂流程。

### 3.3 电商订单是核心入口

3D 打印电商不是先生产再销售，很多情况是：

```text
订单来了 → 看库存 → 有货直接发 → 没货生成生产任务 → 打印 → 发货 → 算利润
```

因此订单详情页必须能直接看到：

```text
商品
SKU
生产状态
发货状态
售后状态
成本明细
最终利润
```

### 3.4 SKU 是核算核心

产品只是归类，SKU 才是：

```text
销售对象
库存对象
生产对象
成本对象
利润对象
模型绑定对象
BOM 绑定对象
```

所有利润、库存、生产、模型、BOM 都应最终落到 SKU。

### 3.5 库存必须走流水

禁止直接改库存数字。  
所有变化都要生成库存流水。

### 3.6 利润必须可追溯

每个订单利润必须能追溯到：

```text
销售金额
平台扣费
广告费
生产成本
耗材批次
包装成本
快递成本
售后成本
补发成本
退款金额
```



```text

```

### 5.5 发货不单独做一级页面

发货集成到：

```text
订单列表
订单详情
批量发货弹窗
```

### 5.6 AI 标题、主图、详情页不单独做一级菜单

AI 电商内容能力集成到：

```text
产品中心 → 产品详情 / SKU 详情 → 电商内容 Tab
产品中心 → AI 内容工作台 Tab
```

原因：标题、主图、详情页都是围绕具体产品和 SKU 产生的，不能脱离产品单独管理。

### 5.7 竞品监控不单独做一级菜单

竞品监控集成到：

```text
产品中心 → 产品详情 → 竞品监控 Tab
产品中心 → 竞品动态 Tab
首页 → 竞品异常提醒
```

每个产品最多添加 5 个竞品链接，用于记录价格、销量展示值、活动、主图、标题变化。

### 5.8 自动选品不单独做一级菜单

自动选品集成到：

```text
产品中心 → 自动选品池 Tab
```

选品结果可以一键转成产品草稿，继续进入产品管理、AI 内容制作、竞品监控、成本测算和生产验证流程。

### 5.9 售后不建议第一版做一级菜单

售后集成到：

```text
订单详情 → 售后 Tab
订单列表 → 售后状态筛选
首页 → 售后待处理
报表 → 售后损失分析
```

如果后期售后量很大，再增加独立售后中心。

## 13A. 电商数据、竞品数据与合规采集原则

本系统新增的竞品监控、价格动态、销量展示值、自动选品功能都依赖电商数据。为了商业化稳定，必须从一开始就把数据来源分级设计。

### 13A.1 数据来源优先级

```text
第 1 优先级：平台开放 API / 服务商 API
第 2 优先级：商家后台导出的 Excel / CSV
第 3 优先级：用户手动添加链接和手动录入竞品信息
第 4 优先级：用户购买的合规第三方数据服务
第 5 优先级：公开页面低频采集，只采集公开展示信息
```

### 13A.2 不能承诺的数据

系统不能承诺一定能拿到：

```text
竞品真实后台销量
竞品真实成交额
竞品真实转化率
竞品广告投放成本
竞品买家信息
平台内部排名权重
```

可以展示为：

```text
销量展示值
销量估算值
价格快照
活动状态
评价数变化
标题 / 主图变化
```

### 13A.3 合规原则

禁止：

```text
绕过平台登录权限
绕过验证码
高频恶意抓取
采集个人信息
破解接口
模拟违规请求
```

系统需要提供：

```text
采集频率限制
失败重试限制
数据来源记录
原始数据快照
采集日志
用户授权记录
合规提示
```

### 13A.4 第一阶段落地策略

第一阶段不要把自动采集作为阻塞项。

建议做法：

```text
竞品链接先允许用户手动添加
竞品价格和销量展示值允许手动录入或导入
```

## 18. 关键业务算法

### 18.1 SKU 预估成本

```text
SKU 预估成本 =
耗材克数 × 当前耗材每克成本
+ 损耗克数 × 当前耗材每克成本
+ 包装成本
+ 人工成本
+ 机器折旧
+ 电费
+ 后处理成本
```

### 18.2 订单毛利

```text
订单毛利 =
实收金额
- 产品生产成本
- 包装成本
- 快递成本
- 平台佣金
- 支付手续费
```

### 18.3 订单净利

```text
订单净利 =
订单毛利
- 广告成本
- 售后成本
- 退款金额
- 补发成本
```

### 18.4 经营净利润

```text
经营净利润 =
订单净利合计
- 固定费用
- 其他费用
```

### 18.5 库存公式

```text
可售库存 = 当前库存 - 锁定库存
预计库存 = 当前库存 + 在产数量 - 待发货数量
```

### 18.6 库存预警

```text
如果 当前库存 <= 库存警戒线
生成库存预警
```

### 18.7 负利润预警

```text
如果 订单净利 < 0
标记为负利润订单
```

### 18.8 未设置成本预警

```text
如果 SKU 没有 BOM 或没有默认耗材
标记为未设置成本 SKU
```

### 18.9 标题评分

```text
标题评分 = 关键词覆盖率 × 35% + 卖点清晰度 × 25% + 平台适配度 × 20% + 点击吸引力 × 10% + 风险词反向分 × 10%
```

### 18.10 内容完整度评分

```text
内容完整度 = 标题完成度 × 20% + 主图完成度 × 25% + 详情页完成度 × 25% + 竞品绑定完成度 × 15% + 成本设置完成度 × 15%
```

### 18.11 竞品价差

```text
竞品最低价 = 绑定竞品当前价格最小值
竞品平均价 = 绑定竞品当前价格平均值
本品价差 = 本品售价 - 竞品平均价
价格压力 = 本品售价 / 竞品最低价
```

### 18.12 自动选品机会评分

```text
机会评分 = 需求热度 × 30% + 毛利空间 × 25% + 竞争友好度 × 20% + 3D 打印适配度 × 15% + 售后风险反向分 × 10%
```

### 18.13 3D 打印适配度

```text
3D 打印适配度 = 结构可打印性 × 30% + 打印时间友好度 × 25% + 耗材成本友好度 × 20% + 后处理复杂度反向分 × 15% + 运输破损风险反向分 × 10%
```

### 18.14 调价建议

```text
如果 本品利润率低 且 竞品价格高于本品
建议提价

如果 本品价格高于竞品最低价过多 且 销量下降
建议优化主图 / 标题，谨慎降价

如果 竞品降价但本品利润率仍健康
建议观察，不自动降价

如果 竞品上活动且本品库存充足
建议设置短期活动或组合装
```
