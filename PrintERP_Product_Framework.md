# PrintERP 3D 打印商家经营管理系统产品开发框架

> 本文档用于放入 Codex / 项目工作目录，作为产品需求、开发方向、数据库设计和业务规则的长期参考文档。

---

## 1. 产品定位

PrintERP 是一个面向 3D 打印产品商家的云端 SaaS 经营管理系统。

目标用户包括：

- 3D 打印网店商家
- 多平台电商卖家
- 线下销售 3D 打印产品的商家
- 小型 3D 打印工作室
- 希望管理订单、库存、生产、利润和售后的商家

核心目标：

- 自动统计每日、每周、每月利润
- 计算每个订单真实利润
- 追踪耗材采购批次和真实成本
- 管理 3D 打印生产任务、失败率和成品库存
- 管理快递、包装、售后、补发、退款等成本
- 支持多个店铺、多个渠道、多个商家使用
- 支持云端部署，通过账号登录使用
- 未来可作为 SaaS 授权给其他商家使用

一句话描述：

> PrintERP 是专为 3D 打印商家打造的云端轻量 ERP，集订单、生产、耗材、库存、售后、财务利润和 SaaS 授权于一体。

## 2. 核心业务闭环

系统主流程：

```text
采购耗材 / 包装材料
→ 入库
→ 建立产品 BOM / 打印配方
→ 接收销售订单
→ 创建生产任务
→ 消耗耗材
→ 成品入库或直接发货
→ 记录快递成本和包装成本
→ 售后处理
→ 自动计算订单利润、日利润、周利润、月利润
```

系统必须围绕以下核心问题设计：

1. 每天、每周、每月到底赚了多少钱？
2. 每个订单是否赚钱？赚多少？
3. 每个 SKU 的真实成本是多少？
4. 哪些产品利润高？哪些产品亏损？
5. 哪些耗材批次成本高、损耗高？
6. 哪台打印机产能高、失败率低？
7. 售后、补发、发错货、打印失败造成了多少损失？
8. 当前库存是否足够？是否需要补货？
9. 多个店铺和线下销售能否统一管理？

## 4. SaaS 多租户设计

系统需要支持多个商家同时使用，每个商家的数据必须完全隔离。

### 4.1 租户概念

每个商家对应一个 tenant。

所有业务数据都必须带 tenantId。

重要字段：

```text
tenantId
createdBy
updatedBy
createdAt
updatedAt
deletedAt
```

### 4.2 用户角色

默认角色：

| 角色 | 权限说明 |
|---|---|
| 平台超级管理员 | 管理所有商家、套餐、系统配置 |
| 商家老板 | 查看和管理本商家全部数据 |
| 店长 | 管理订单、库存、生产、售后 |
| 生产员 | 查看和处理生产任务 |
| 仓库员 | 入库、出库、盘点、发货 |
| 财务 | 查看成本、利润、费用、报表 |
| 客服 | 处理订单售后、补发、退款记录 |

### 4.3 权限原则

- 不同商家之间数据完全隔离
- 员工只能访问所属商家的数据
- 财务数据默认只给老板和财务角色查看
- 重要操作需要记录操作日志
- 删除数据建议使用软删除 deletedAt

## 6. 采购与耗材批次成本管理

3D 打印耗材每次采购的价格、颜色、材质、品牌都可能不同，因此必须按批次管理。

### 6.1 采购单功能

功能：

- 新增采购单
- 选择供应商
- 采购耗材或包装材料
- 录入采购数量
- 录入采购金额
- 录入采购运费
- 录入税费
- 录入优惠金额
- 自动计算入库总成本
- 自动计算每克成本
- 采购入库后生成库存流水

### 6.2 成本公式

```text
入库总成本 = 采购金额 + 采购运费 + 税费 - 优惠金额

每克成本 = 入库总成本 ÷ 入库克数
```

### 6.3 耗材批次字段

```text
批次编号
供应商
材质
颜色
品牌
采购日期
采购数量/g

采购运费
税费
优惠金额
最终入库成本
每克成本
当前剩余克数
状态：正常 / 用完 / 报废
```

### 6.4 耗材成本出库方式

MVP 阶段建议：移动平均成本。

第二阶段支持：FIFO 先进先出。

## 7. 产品 BOM / 打印配方

每个 SKU 需要设置一套默认生产配方，用于预估成本和创建生产任务。

### 7.1 BOM 内容

字段：

```text
SKU

理论耗材克数
支撑 / 损耗克数
预计打印时间
后处理材料
包装材料
人工分钟数
默认打印机类型
备注
```

### 7.2 产品理论成本公式

```text
理论材料成本 = 理论耗材克数 × 当前耗材每克成本
理论损耗成本 = 损耗克数 × 当前耗材每克成本
理论包装成本 = 包装材料成本合计
理论人工成本 = 人工分钟数 × 每分钟人工成本
理论机器折旧 = 打印小时数 × 打印机每小时折旧成本
理论产品成本 = 材料成本 + 损耗成本 + 包装成本 + 人工成本 + 机器折旧 + 电费
```

## 9. 库存管理

库存分为三大类：

1. 耗材库存
2. 包装材料库存
3. 成品库存

### 9.1 库存原则

禁止直接修改库存数字。

所有库存变化必须生成库存流水。

库存流水类型：

```text
采购入库
生产消耗
生产入库
销售出库
售后补发出库
退货入库
盘盈
盘亏
报废
手动调整
```

### 9.2 库存公式

```text
可售库存 = 当前库存 - 已锁定库存

预计库存 = 当前库存 + 在产数量 - 待发货数量

如果 当前库存 <= 库存警戒线，则生成库存预警
```

### 9.3 库存预警

预警类型：

```text
耗材低库存
包装材料低库存
成品低库存
高库存积压
待生产超时
待发货超时
```

## 18. 推荐技术架构

MVP 推荐技术栈：

```text
前端 + 后端：Next.js + TypeScript
数据库：PostgreSQL
ORM：Prisma
UI：Tailwind CSS + shadcn/ui
认证：Auth.js 或自定义 JWT
图表：Recharts 或 ECharts
Excel / CSV 导入：papaparse、xlsx
部署：Docker Compose
反向代理：Nginx 或 Caddy
缓存 / 队列：Redis，第二阶段再加入
```

架构建议：

- 第一版使用模块化单体
- 所有业务模块放在同一个项目中
- 数据库使用 PostgreSQL
- 金额字段必须使用 Decimal
- 库存数量和耗材克数使用 Decimal
- 所有库存变化必须使用事务处理
- 关键财务和库存操作必须写入 audit_logs

## 22. 关键业务规则

### 22.1 成本必须可追溯

每一笔成本都要能追溯到：

```text
订单
SKU
生产任务
耗材批次
打印机
发货记录
售后记录
费用记录
```

### 22.2 库存必须走流水

禁止直接改库存。

所有库存变化必须生成 inventory_transactions。

### 22.3 利润必须自动重算

以下情况发生时，订单利润必须重新计算：

```text
订单金额变化
产品成本变化
生产任务完成
发货成本变化
包装成本变化
平台手续费变化
广告成本变化
售后发生
退款发生
补发发生
```

### 22.4 金额字段不能用 float

所有金额字段必须使用 Decimal。

错误做法：

```text
float
number 直接计算金额
```

正确做法：

```text
Decimal
Prisma Decimal
数据库 numeric / decimal
```

### 22.5 多租户数据隔离

所有查询必须带 tenantId。

禁止跨租户读取数据。
