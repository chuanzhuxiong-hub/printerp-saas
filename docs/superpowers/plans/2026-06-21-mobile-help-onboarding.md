# 帮助中心与新手引导手机端优化计划

## 目标

在不改动数据库结构和业务逻辑的前提下，优化帮助中心、新手入门和初始化向导在手机端的阅读与操作体验。

## 范围

- 帮助中心首页 /app/help
- 新手入门 /app/help/getting-started
- 通用帮助主题页组件
- 初始化向导组件
- 步骤卡片组件

## 不做

- 不修改 Prisma schema
- 不修改 onboarding 状态判断逻辑
- 不新增一级菜单
- 不调整订单、库存、利润、权限等高风险业务逻辑

## 执行步骤

1. 优化初始化进度卡片在手机端的间距、按钮宽度和项目网格。
2. 修复 StepGuide 完成符号乱码，并优化步骤卡片的触控尺寸。
3. 优化帮助主题页文章、教程表格、注意事项块的移动端间距。
4. 优化帮助中心首页和新手入门页面的移动端卡片布局。
5. 运行 TypeScript 和生产构建验证。

## 风险

本阶段仅修改 Tailwind class 和静态 UI 结构，风险较低。主要风险是响应式 class 影响桌面端排版，需要通过构建和页面抽查验证。
