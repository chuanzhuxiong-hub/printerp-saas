# 重业务页面手机端补漏计划

## 目标

对产品中心、订单中心、库存中心、采购中心和订单导入页做手机端真实使用补漏，提升筛选、分页、按钮和详情页的可操作性。

## 范围

- 通用组件：PageHeader、FilterBar、DataTable
- 产品中心：Tabs、筛选栏、SKU 展开区、分页
- 订单中心：操作按钮、筛选栏、分页
- 库存中心：操作按钮、筛选栏
- 采购中心：快捷入口、采购详情页
- 订单导入页：页面结构和手机端按钮布局

## 不做

- 不修改 Prisma schema
- 不修改订单导入解析规则
- 不修改库存流水、采购入库、利润计算和 tenantId 查询
- 不拆分 SKU、BOM、发货、售后为一级菜单

## 风险

本阶段以 Tailwind class 和展示文案为主，业务风险较低。采购详情页会重写 JSX 展示结构，但继续使用原有查询和原有接口动作。

## 验证

- npx.cmd tsc --noEmit
- npm.cmd run build
- 部署后检查 /api/health
